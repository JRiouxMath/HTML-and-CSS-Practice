Hyperlinks Notes

I don't really feel like getting into this right now, but it might be good to have it here as a reference. For further reference, go back to the Mozilla tutorial and learn more.

The root is the folder that all the files are kept in for the webpage. A directory is a folder within the root. This is good for keeping things together like pdfs or data files.

If you are linking to something in the same directory, just give the filename. To go down a directory, use directory_name / file_name. To go up a directory, use "../" and file_name. These can be chained as needed. These are considered relative URLs.

To link to something outside your page you must use the full URL. This is an absolute URL.

Common practices include alerting the user when a link will involve special circumstances including downloading, streaming requiring a Flash player or leaving a secured site.

It is also possible to link to an email server with special cases such as including cc, bc and attachments or using a specific server.

Abbreviations

Since we love abbreviating things, we have to be aware that not everyone will know what the abbreviation or achronym stands for. HTML has an element to address this. Originally they created one for each case, but they are so similar that the achronym one is no longer supported and rarely used. Use <abbr> in either case. This will add a tooltip that will show the full word title.
